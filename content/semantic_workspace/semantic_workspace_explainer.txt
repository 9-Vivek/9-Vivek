I’m building an open-core product called Semantic Workspace. The core idea is that codebases should not be navigated only through folder paths. I want to add a semantic layer above files so developers and AI agents can understand systems by purpose, feature, layer, ownership, relationships, and architectural intent.

The roadmap starts with defining the semantic foundation. I need a typed metadata schema with fields like feature, layer, purpose, runtime, owner, related files, and agent notes. The metadata should live directly in the repository inside a Git-friendly `.semantic-workspace/` directory containing schemas, file metadata, saved views, and exported context bundles. Early workflows are intentionally manual because manual annotation forces teams to make architectural intent explicit and helps validate which fields are actually useful.

The first product version is a local developer tool centered around a VS Code extension. The extension should read semantic metadata, display files grouped by concepts like feature or layer, and support saved semantic views. I also want a semantic explorer that lets developers navigate the repo through different projections instead of raw folder trees. Another core capability is exporting structured context bundles for AI coding agents so they receive concise explanations of relevant files, relationships, risks, and tests instead of raw file dumps.

After the MVP exists, the next step is validating whether semantic navigation and semantic AI context actually improve developer workflows. I want to test the system on real repositories, measure coding-agent performance with and without semantic context bundles, and interview developers about onboarding, architecture confusion, code review, and AI-agent mistakes. The goal is to confirm that developers are willing to maintain metadata because the resulting workflows are genuinely useful.

The project is intended to become open source with an open-core business model. The free layer includes the schema, local storage format, extension, CLI tooling, semantic views, and context export. The schema itself is important because it could become a shared interoperability layer that other tools can consume. I also want CLI tooling for validation, automation, CI workflows, and agent integrations, plus example repositories showing different semantic workspace setups.

Once manual workflows are validated, I want to add AI-assisted metadata suggestions and relationship inference. AI should suggest metadata and relationships while humans remain in control. The system should eventually infer connections like imports, tests, routes, or commonly co-edited files. From there, the product can generate task-aware context bundles where a user describes a task and the system recommends relevant files, summaries, risks, and tests. The product is not meant to replace coding agents like Claude Code or Cursor; it is meant to become the context layer that improves them.

The roadmap then expands into team workflows. I want shared semantic views, ownership mapping, semantic pull request summaries, and architecture drift detection. The semantic layer should become collaborative infrastructure for onboarding, reviews, planning, and lightweight governance.

Commercialization happens through team and enterprise features rather than locking the core schema behind a paywall. Paid functionality includes dashboards, collaboration workflows, integrations, managed AI features, governance controls, and enterprise tooling. Over time, the product can expand into organization-level semantic graphs spanning multiple repositories, onboarding maps, and AI coding governance systems that control risk, ownership, and review expectations.

Long term, I think the idea extends beyond code. The deeper concept is a semantic layer for files in general. Code is the initial wedge because AI-assisted development creates a strong need for structured context, but similar problems exist in research, legal work, enterprise documentation, and knowledge management. The broader vision is infrastructure that helps humans and AI systems understand, navigate, and safely operate on complex information systems through structured semantic meaning rather than folder hierarchies alone.
